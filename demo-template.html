Tabbed Navigation

Tabs let learners explore related content at their own pace without overwhelming them with everything at once. Best for categorised content where order doesn't matter — tools, departments, product features.

Best for
Categorised content, reference material, comparing options
Bloom's Level
L1–L2 · Remember / Understand
Storyline Setup
One base layer + separate content layers. Variables control which layer is visible. JavaScript fires SetVar on click.

GDPR — General Data Protection Regulation
GDPR governs how personal data about EU citizens is collected, stored, processed, and deleted. Every employee who handles customer data has direct compliance obligations under GDPR — not just the data protection team. Key obligations: lawful basis for processing, data minimisation, right to erasure, breach notification within 72 hours.
AML — Anti-Money Laundering
AML regulations require financial institutions to detect, report, and prevent money laundering. Key requirements include Customer Due Diligence (CDD), Suspicious Activity Reports (SAR) filed within 30 days, and transaction monitoring. Every employee in a client-facing role must understand the red flags and escalation process.
Data Security
Data security policies govern how digital information is protected against unauthorised access, disclosure, modification, or destruction. For employees, this means: using approved devices, not sharing credentials, encrypting sensitive data in transit, and reporting suspicious activity within 24 hours. The biggest risk is human error — not sophisticated attacks.
Code of Conduct
The Code of Conduct defines the behavioural and ethical standards expected of every employee. It covers conflicts of interest, gifts and entertainment limits, workplace behaviour, social media use, and whistleblowing. When in doubt: if you wouldn't be comfortable seeing it on the front page of a newspaper, don't do it.
JavaScript — Storyline Tab Logic
// Tab switching in Storyline — runs on each tab button click
var player = GetPlayer();

// Set which tab is active (1–4)
function activateTab(tabNumber) {
  player.SetVar("ActiveTab", tabNumber);
  
  // Hide all layers, show the selected one
  [1, 2, 3, 4].forEach(function(n) {
    player.SetVar("ShowTab" + n, n === tabNumber);
  });
}

// Call from each tab button's Execute JavaScript trigger:
activateTab(1); // Tab 1 button
activateTab(2); // Tab 2 button

Expandable Accordion

Accordions let learners choose what to explore without scrolling through walls of text. Perfect for FAQ sections, step-by-step procedures, or glossaries where not everything needs to be read.

Best for
Procedures, FAQs, reference content, optional detail
Bloom's Level
L1–L3 · Remember to Apply
Accessibility Note
Headers must have role="button", tabIndex=0, and respond to Enter/Space. See Accessibility Guide for full implementation.

What triggers a Suspicious Activity Report (SAR)?+
A SAR must be filed when you have reasonable grounds to suspect that funds are proceeds of criminal activity or that a transaction may be connected to money laundering or terrorist financing. This includes unusual transaction patterns, inconsistent customer behaviour, or transactions with no apparent business purpose. You must file within 30 days of suspicion arising.
What is the difference between CDD and EDD?+
Customer Due Diligence (CDD) is the standard verification process applied to all customers — identity verification, understanding the business relationship, and ongoing monitoring. Enhanced Due Diligence (EDD) applies additional scrutiny to high-risk customers: Politically Exposed Persons (PEPs), customers in high-risk jurisdictions, or relationships with complex structures. EDD requires more evidence and senior management approval.
Can I tell a customer they are being investigated?+
No. This is known as "tipping off" and is a criminal offence under AML legislation. If you suspect money laundering and have filed or are considering filing a SAR, you must not disclose this to the subject of the investigation or any third party. Continue acting normally with the customer until directed otherwise by your compliance team.
What are the penalties for non-compliance?+
Penalties for AML non-compliance range from regulatory fines (into the billions for major institutions) to criminal prosecution of individual employees. Regulators can also impose business restrictions, licence suspension, or require remediation programmes costing far more than the original fine. Personal liability for compliance officers is increasing year-on-year.
JavaScript — Accordion Toggle
// Accessible accordion toggle for Storyline
function toggleAccordion(headerEl) {
  var player = GetPlayer();
  var body = headerEl.nextElementSibling;
  var isOpen = headerEl.classList.contains("open");
  
  // Close all, open clicked one
  document.querySelectorAll(".accordion-header").forEach(function(h) {
    h.classList.remove("open");
    h.setAttribute("aria-expanded", "false");
    h.nextElementSibling.style.maxHeight = "0";
  });
  
  if (!isOpen) {
    headerEl.classList.add("open");
    headerEl.setAttribute("aria-expanded", "true");
    body.style.maxHeight = body.scrollHeight + "px";
    player.SetVar("OpenSection", headerEl.textContent);
  }
}

// Add keyboard support
document.querySelectorAll(".accordion-header").forEach(function(h) {
  h.addEventListener("keydown", function(e) {
    if (e.key === "Enter" || e.key === " ") {
      e.preventDefault();
      toggleAccordion(e.target);
    }
  });
});

Branching Scenario

Branching scenarios are the most effective format for compliance and behaviour-change training. They put learners inside realistic decisions with real consequences — the only format that tests Apply (Bloom's L3) rather than Remember.

Best for
Compliance, sales conversations, difficult situations, behaviour change
Bloom's Level
L3–L4 · Apply / Analyse
ID Model Note
Pair with Action Mapping — every branch represents a real on-the-job decision. Consequences should be realistic, not cartoon.

JavaScript — Branching Logic
// Branching scenario answer handler
var player = GetPlayer();
var correctAnswer = 1; // B is correct

function handleAnswer(selectedIndex) {
  var correct = selectedIndex === correctAnswer;
  
  // Record the attempt
  player.SetVar("SelectedAnswer", selectedIndex);
  player.SetVar("AnswerCorrect", correct);
  player.SetVar("ShowFeedback", true);
  
  if (correct) {
    player.SetVar("Score", player.GetVar("Score") + 1);
    // Route to next scenario or success layer
    player.GotoSlide("Scenario 2");
  } else {
    // Show consequence layer, then allow retry or route forward

}

Drag & Drop — Accessible Build

Drag and drop adds interactivity but is the most accessibility-challenging interaction in eLearning. This implementation includes full keyboard fallback — every action performable with Tab, Enter, and Arrow keys.

Best for
Categorisation, ordering, matching, sorting tasks
Bloom's Level
L2–L3 · Understand / Apply
Accessibility Note
Keyboard mode: Tab to chip, Enter to pick up, Tab to target, Enter to drop. See the JS Snippets library for the full accessible implementation.

JavaScript — Accessible Drag & Drop
// Accessible drag and drop with keyboard fallback
var player = GetPlayer();
var dragItem = null;
var keyboardMode = false;
var selectedChip = null;

// Mouse drag events
function onDragStart(e) {
  dragItem = e.target;
  e.dataTransfer.setData("text/plain", dragItem.dataset.cat);
}

function onDrop(e, targetZone) {
  e.preventDefault();
  if (dragItem) {
    targetZone.appendChild(dragItem);
    player.SetVar("ItemPlaced", dragItem.dataset.id);
    checkCompletion();
  }
}

// Keyboard fallback — Tab to navigate, Enter to pick/drop
document.querySelectorAll(".dnd-chip").forEach(function(chip) {
  chip.tabIndex = 0;
  chip.addEventListener("keydown", function(e) {
    if (e.key === "Enter") {
      keyboardMode = true;
      selectedChip = chip;
      chip.setAttribute("aria-grabbed", "true");
      // Move focus to drop targets
      document.querySelector(".dnd-target").focus();
    }
  });
});
