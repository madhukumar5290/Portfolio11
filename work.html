Compliance · Banking · End-to-End Course

The Challenge

Compliance training in banking is typically dry, slide-heavy, and forgotten within days of completion. The challenge was to make AML regulations genuinely engaging — and to ensure learners could apply the knowledge to real-world detection scenarios, not just recall definitions. Regulatory frameworks require pass scores; they do not require learners to actually understand how money laundering works.

ID Model Decision

Action Mapping (Cathy Moore) was applied first: I started by identifying the business goal (reduce compliance violations) and mapped the specific decisions compliance officers need to make — not the information they need to memorise. This stripped roughly 40% of the original content brief.

Bloom's Taxonomy Level 3 (Apply) drove the assessment design. Rather than asking "What does CTF stand for?" — a Level 1 recall question — every assessment asks learners to evaluate a scenario and decide what a compliance officer should do next.

"The design decision was to put learners inside a suspicious transaction review, not outside it reading about one. Every branching path leads to a consequence that explains why it was right or wrong — not just a score."

What I Built

End-to-end: storyboard, visual design, character-based branching interactions, regulatory-grounded feedback, and SCORM 2004 packaging. The course uses a storyline arc where the learner is promoted to compliance analyst and must review three escalating cases — each more complex than the last.

ADA/508WCAG 2.1 AAJavaScriptStoryline 360
ID Model Applied

Launch Demo →
The Problem with Storyline Drag-and-Drop

Articulate Storyline's default drag-and-drop interaction is completely inaccessible to keyboard users. Tab order defaults to object creation sequence — not logical reading order. There is no keyboard activation. There is no visual focus indicator. Screen readers announce nothing useful. The entire interaction is locked out for approximately 15–20% of potential learners.

The UDL Framework Decision

Universal Design for Learning (UDL) requires that learning experiences provide multiple means of action and expression. A drag-and-drop interaction that only works with a mouse fails this principle at the architectural level — before a single slide is published.

The solution: rebuilt the interaction with custom JavaScript to enforce logical tab order, keyboard activation via Enter and Space, visible focus indicators meeting WCAG 2.1 AA contrast requirements, and screen reader announcements via ARIA live regions. Accessibility was the architecture — not a patch applied after the fact.

"Most developers add alt text at the end and call it done. Real accessibility means the tab order, focus management, and keyboard architecture are designed into the first slide master — before any content is built."

What Was Built

A reusable Storyline 360 interaction template with full keyboard operability, tested against NVDA, JAWS, and VoiceOver. The JavaScript layer manages focus, announces state changes via ARIA, and provides identical functionality for pointer and keyboard users. This template is now a reusable asset for all ADA-required courses.

01 · Compliance · Banking

Anti-Money Laundering

ID Model Applied
ADDIE + Action Mapping (Cathy Moore) · Bloom's Taxonomy Level 3 (Apply) · Kirkpatrick Level 3 Evaluation Design
The Challenge

AML compliance training is typically a slide deck of regulations that learners click through to get a completion stamp. The business goal — reducing actual compliance violations — requires learners to know how to detect suspicious activity and what to do about it, not to recite definitions under exam conditions.

ID Decisions

Action Mapping stripped the content list from 47 slides to 28 by asking: which of these pieces of information support a specific action the learner needs to take? The remaining content was restructured around decision moments, not topic categories.

Bloom's Level 3 (Apply) drove assessment design. Every knowledge check asks learners to evaluate a transaction scenario and decide what a compliance officer should do — not what CTF stands for.

What Was Built

Three escalating branching scenarios. Learner is positioned as a newly promoted compliance analyst. Each scenario involves reviewing documentation, interviewing a client, and making a regulatory decision. Feedback is immediate, specific, and tied to the actual regulation being applied. SCORM 2004 packaged.

02 · Accessibility · ADA/508

Drag & Drop with Full Keyboard Accessibility

ID/Design Framework
Universal Design for Learning (UDL) · WCAG 2.1 Success Criteria 1.3.1, 2.1.1, 2.4.3 · Section 508
The Accessibility Problem

Storyline's default drag-and-drop interaction is inaccessible by design. Tab order defaults to object creation order. There is no keyboard activation. Screen readers announce "button" with no context. Approximately 15–20% of learners — those using keyboard navigation, screen readers, or assistive switches — are completely locked out.

UDL Framework Decision

Universal Design for Learning Principle 2 requires multiple means of action and expression. A mouse-only interaction violates this at the architectural level. The solution: custom JavaScript enforcing logical tab order, Enter/Space keyboard activation, ARIA live region announcements, and visible focus indicators at a 3:1 contrast ratio. Accessibility was the architecture — not a patch.

Technical Implementation

JavaScript layer manages DOM focus, state announcements via ARIA roles, keyboard event handling (Tab, Enter, Space, arrow keys for drop target selection), and success/error state communication. Tested against NVDA, JAWS, and VoiceOver. Reusable template now available for all ADA-required courses.

The Conversion Challenge

Flash's end-of-life created a crisis across L&D libraries worldwide. The challenge in conversion is not just technical — it's preserving the original learning intent and interaction fidelity while adapting for HTML5 delivery on devices that didn't exist when the course was built.

04 · Game-Based Learning · Science

Bees and Beehives — Interactive Game

ID Model Applied
Merrill's First Principles of Instruction · Game-Based Learning Design · Bloom's Level 2–3
Merrill's Framework in Practice

Rather than opening with "here is information about bee biology," the course opens with a real-world problem: a beekeeper's colony is mysteriously failing. Learners investigate through exploration — discovering biology facts through consequence rather than exposition. This is Merrill's Problem-Centred principle in direct practice.

Activation (connecting to what learners already know about insects), Demonstration (showing bee behaviours through animation), Application (learners make colony management decisions), and Integration (connecting to environmental impact) are all present — not as a checklist, but as a natural game narrative arc.

05 · Game-Based Learning · Cognition

Memory Flash Cards — Desktop

ID Model Applied
Cognitive Load Theory (Sweller) · Spaced Repetition (Ebbinghaus) · Retrieval Practice Effect
The Science Behind the Game

Memory flash cards are not just a game mechanic — they implement two of the most evidence-backed principles in learning science. The Retrieval Practice Effect demonstrates that actively recalling information from memory strengthens memory traces far more effectively than re-reading. Spaced Repetition schedules those retrieval attempts at increasing intervals to fight the Forgetting Curve.

The game is designed to be adaptable to any subject matter — the card content is templated and the mechanics remain constant. This makes it a reusable asset for post-training knowledge reinforcement across any topic.

07 · Organisational Induction · Non-Profit

About World Vision India

ID Model Applied
Gagné's 9 Events of Instruction · ADDIE · Values-Driven Narrative Design
Gagné's Framework in Induction Design

Organisational induction is one of the highest-stakes training events in the employee lifecycle. First impressions of organisational culture, values and mission are formed here — and they're sticky. The design must earn attention (Event 1), connect to why the learner chose this organisation (Event 3 — prior activation), and leave them with a genuine sense of purpose (Event 9 — integration).

Every one of Gagné's nine events is explicitly mapped to a screen or interaction in this course — from the opening statistic about children served (gain attention) through to the reflection prompt about what the learner's role means in the context of WVI's mission (enhance retention and transfer).

08 · Leadership Development · Innovation

Solution Innovation Program

ID Model Applied
ADDIE · Bloom's Taxonomy Levels 4–5 (Analyse & Evaluate) · Merrill's Problem-Centred
High-Level Cognitive Design

Innovation training fails when it teaches innovation frameworks without requiring learners to actually innovate. The design brief was ambitious: move beyond Level 2 (Understand) and Level 3 (Apply) to genuinely assess Level 4 (Analyse) and Level 5 (Evaluate) — asking learners to critique existing solutions, identify root causes, and defend a proposed innovation against structured critique.

Merrill's Problem-Centred principle anchors the course: each module begins with a real organisational problem (presented as a case study with actual data) rather than a framework definition. Learners encounter the problem before they receive the tools to solve it — creating the cognitive need that makes the learning relevant.

06 · Puzzle Game · Cognitive

Number Puzzle Game

ID Model Applied
Cognitive Load Theory · Puzzle-Based Learning Design
The Design Problem

Number fluency drills are usually flashcards or timed drills — effective for speed, poor for engagement. The brief asked for a format that built working-memory capacity through repeated number manipulation without feeling like a worksheet.

ID Decision

Cognitive Load Theory shaped the puzzle difficulty curve: each level introduces exactly one new constraint (a new operation, a new sequencing rule) rather than several at once, keeping intrinsic load manageable while extraneous load — decorative noise, unclear instructions — was deliberately minimised.

What Was Built

A progressive number-sequencing puzzle with 12 levels, an in-built hint system that scaffolds rather than solves, and a scoring model that rewards accuracy over speed — directly counteracting the "rush and guess" behaviour common in timed numeric games.

The Challenge

Translating the English geometry module's text was the easy part. The harder requirement: a true right-to-left experience where the interaction itself — not just the labels — reads naturally for Arabic-speaking learners, including the direction shapes transform in.

ID Decision

Rather than simply mirroring the English layout, the entire slide master was rebuilt with RTL-native alignment, Arabic typography sized for legibility at the same visual weight as the Latin original, and the geometric animation direction reviewed with a native-speaking reviewer to confirm it matched RTL reading expectations.

The Challenge

Financial literacy content is usually delivered as advice — "you should budget like this" — which learners nod along to and then ignore, because they've never felt the actual trade-off of a bad financial decision in a safe environment.

20 · Mobile · Responsive · Memory

Memory Flash Cards — Mobile

ID Model Applied
Spaced Repetition · Mobile-First UX Design
The Challenge

The original Memory Flash Cards build was desktop-first, with click targets and hover states that translate poorly to touch. Learner analytics showed a meaningful share of attempted mobile sessions, with high drop-off — a direct signal the experience needed a genuine mobile-first rebuild, not a responsive patch.

ID Decision

Rather than scaling the desktop layout down, the interaction model was redesigned around thumb-reachable tap zones and a portrait-first card flow, while preserving the identical spaced-repetition logic that makes the underlying learning mechanic effective.

What Was Built

A touch-optimised flash card experience with swipe-based card progression, larger tap targets sized to mobile accessibility guidelines, and session state that persists if a learner is interrupted mid-session — a common mobile usage pattern the desktop version didn't need to handle.
