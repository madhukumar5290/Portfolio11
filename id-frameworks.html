ID Cheat Sheet

Every ID framework,
clearly explained.

A practical reference for ID models I apply in real projects — explained as design decisions with when to use each and when not to. Now includes 10 frameworks.

ADDIE
Analysis · Design · Development · Implementation · Evaluation

The industry-standard instructional design framework. A five-phase process providing structure without sacrificing flexibility. Best for projects with clear scope, defined SMEs, and stakeholders who need a documented process trail for compliance or audit.

Best For
Formal corporate eLearning, regulated industries, large-scale deployments with multiple stakeholders
Avoid When
Rapidly changing content, agile environments, stakeholders who can't specify requirements upfront
Used In My Work
Anti-Money Laundering, Code of Conduct, World Vision India
A
Analysis
Identify the performance gap, audience characteristics, learning environment, existing knowledge, and constraints. The most important phase — most design failures start with skipping it.
D
Design
Define learning objectives using Bloom's verbs, assessment strategy, content sequence, and media approach. This phase produces the blueprint the development team builds from.
D
Development
Build the actual course — storyboards, interactions, media assets, SCORM packaging. Iterative SME review cycles happen here. Alpha → Beta → Gold master.
I
Implementation
Deploy to LMS, train facilitators if blended, communicate to learners. Chronically underplanned and over-relied upon when earlier phases were weak.
E
Evaluation
Measure against learning objectives. Did learners gain knowledge? Did behaviour change? Kirkpatrick's model is the companion framework — design evaluation from Level 4 backwards.
Bloom's
Bloom's Taxonomy · Revised Anderson & Krathwohl 2001

The framework for writing learning objectives that actually mean something. Six cognitive levels from Remember to Create — each requiring fundamentally different instructional approaches and assessment methods. Used in every single course design decision I make.

Best For
Writing learning objectives, designing aligned assessments, setting cognitive demand levels
Key Insight
Most corporate training targets Level 1–2. Most job performance requires Level 3–4. This gap is where training fails.
1
Remember
Recall facts. Verbs: define, list, recall, identify. Assessment: multiple choice, matching. Most compliance training stops here — necessary but rarely sufficient for performance change.
2
Understand
Explain ideas. Verbs: describe, explain, summarise. Assessment: short answer, concept mapping. The learner can explain it — but cannot yet use it reliably on the job.
3
Apply
Use information in new situations. Verbs: use, demonstrate, execute, solve. Assessment: scenarios, simulations. This is where learning becomes useful on the job.
4
Analyse
Break down and draw connections. Verbs: differentiate, organise, compare. Assessment: branching scenarios with multiple valid approaches, root cause analysis tasks.
5
Evaluate
Justify decisions. Verbs: judge, critique, defend, recommend. Assessment: structured reflection, recommendations with written rationale required.
6
Create
Produce original work. Verbs: design, construct, develop, formulate. Assessment: project work, portfolios, performance tasks. The highest cognitive demand — rarely targeted in typical eLearning.
SAM
Successive Approximation Model · Michael Allen

Michael Allen's agile alternative to ADDIE. Uses rapid prototyping and iterative design cycles instead of completing one phase before starting the next. The best model when timelines are tight or requirements evolve during development.

Best For
Agile environments, SMEs who give better feedback on prototypes than documents, fast-turnaround projects
Key Advantage
Failures happen in the prototype stage (cheap) not the finished published course (expensive).
1
Preparation
Brief background research — just enough to prototype. Don't over-specify. The goal is a working prototype, not a 40-page design document.
2
Iterative Design
Rapid prototype → review → repeat. Stakeholders react to something tangible, not a document. This closes the feedback gap that plagues waterfall projects.
3
Iterative Development
Design, develop, and review in overlapping cycles. Alpha → Beta → Gold master. Each cycle produces a working, reviewable asset.

When I Use SAM

  • Client cannot articulate requirements until they see something
  • Content will change during development
  • Timeline doesn't allow sequential ADDIE phases
  • SMEs are reactive reviewers rather than proactive specifiers
Gagné
Gagné's Nine Events of Instruction

Robert Gagné's nine-step sequence based on cognitive processing theory. Each event maps to a specific internal learning condition. The most useful framework for structuring a single lesson — especially procedural or concept-based content.

Best For
Single-lesson structure, procedural training, ensuring no cognitive step in the learning process is skipped
Key Insight
The nine events are cognitive checkpoints — miss one and learners feel the gap even if they can't name it.
1
Gain Attention
Hook the learner immediately with something that makes them need to know this. If you lose them here, the rest of the course is wasted.
2
State the Objective
Tell learners exactly what they'll be able to do. Clarity removes the extraneous cognitive load spent wondering "why am I doing this?"
3
Stimulate Prior Learning
Connect new content to what learners already know. New knowledge anchors more effectively when tied to existing schema.
4
Present Content
Deliver new knowledge — chunked, sequenced, and scaffolded to avoid cognitive overload. The most over-designed phase in most courses.
5
Provide Guidance
Worked examples, analogies, job aids. Show learners how to use the new knowledge before asking them to use it alone.
6
Elicit Performance
Practice. The learner applies the skill in a scenario, simulation, or exercise. No practice = no learning. This is not optional.
7
Provide Feedback
Immediate, specific, corrective feedback. Not "Incorrect, try again" — but why it's wrong and what to do differently. Feedback is instruction.
8
Assess Performance
A formal check: can the learner perform independently? Must match the objective — same cognitive level, same performance context.
9
Enhance Retention
Spaced practice, job aids, follow-up activities. Consistently the most skipped step — and consistently the reason training doesn't transfer.
Merrill
Merrill's First Principles of Instruction

David Merrill's five principles derived from reviewing the best ID models across decades. The core insight: learning is promoted when instruction is centred on real-world problems, not isolated content coverage. Particularly effective with experienced learners who disengage from courses that re-teach what they know.

Best For
Complex skill development, experienced learners, performance-critical training, problem-solving courses
Used In My Work
Bees & Beehives, Solution Innovation Program, Body Organs, The Money Game
1
Problem-Centred
Open with a real-world problem, not an overview slide. "Here is the problem you need to solve" creates cognitive need before supplying the tools to solve it.
2
Activation
Connect to relevant prior experience. Validate what learners already know before introducing new content — don't teach what they already know.
3
Demonstration
Show the skill being applied in context — not described abstractly. Worked examples are demonstrations. Telling is not demonstrating.
4
Application
Guided practice progressing to independent practice. Corrective, informative feedback is essential — not just correct/incorrect binary scoring.
5
Integration
Connect explicitly to their actual work context. Reflection, peer discussion, or on-the-job application consolidates and makes the learning permanent.
Kirk-
patrick
Kirkpatrick's Four Levels of Training Evaluation

The most widely used evaluation framework in L&D. The critical design principle: design evaluation from Level 4 backwards, not measure from Level 1 forwards. Start with the business result, work backwards to what training must produce.

Best For
Demonstrating L&D value, measurement strategy design, ROI conversations with business stakeholders
Key Insight
Most organisations only measure Level 1. Level 3 and 4 are where real value and accountability live.
1
Reaction
Did learners like it? Satisfaction surveys, smile sheets. Most measured, least useful predictor of performance change. Necessary but not sufficient.
2
Learning
Did learners gain knowledge or skill? Pre/post assessments, skills demonstrations. The level most course designers focus on.
3
Behaviour
Did learners apply new knowledge on the job? Manager observations, 360 feedback, performance monitoring. Rarely measured — the biggest gap in most L&D programmes.
4
Results
Did training produce the intended business outcome? Error reduction, sales increase, safety incidents. The conversation executives actually care about.
Action
Mapping
Cathy Moore's Action Mapping Methodology

Starts by asking "what do people need to DO?" rather than "what do they need to KNOW?" Ruthlessly strips information-delivery in favour of practice activities. The best antidote to content-heavy, click-next eLearning. I hold Cathy Moore's official certification.

Best For
Any project where the client's instinct is to dump information into slides. All compliance training. Performance-gap projects.
Personal Note
First framework I reach for on every new project. Prevents building the wrong course entirely.
1
Identify the Business Goal
Not "we need training on X" — but "we need to reduce violations by 30%." If there's no measurable business goal, there's no justified training project.
2
Map the Actions
What specific behaviours need to change on the job? Map what people need to DO differently — not what they need to know. This eliminates most content the client wanted.
3
Design Practice Activities
For each action, design a realistic practice scenario — not a knowledge check. Learners practise doing, not practise recalling.
4
Minimum Necessary Information
Add only information learners need to complete the practice activities. Typically eliminates 40–60% of the original content list. Everything else is noise.
UDL
Universal Design for Learning · CAST Framework

UDL is a proactive accessibility and inclusion framework based on neuroscience research. Rather than retrofitting accessibility onto finished courses, UDL builds flexible, barrier-free learning from the first design decision. It's how I approach every ADA/508-compliant project.

Best For
Any project requiring ADA/508 compliance, diverse learner populations, K12 and higher education, global deployment
Key Principle
Design for the edges and the middle takes care of itself. If it works for screen readers, it works for everyone.
I
Multiple Means of Representation
Provide information in multiple formats — text, audio, visual, interactive. No single format works for all learners. Captions, transcripts, alt text, and visual alternatives are not add-ons — they are the design.
II
Multiple Means of Action & Expression
Give learners multiple ways to interact and demonstrate learning. Mouse-only interactions fail keyboard users. Single-mode assessments exclude alternative expression. Drag-and-drop must have keyboard alternatives.
III
Multiple Means of Engagement
Provide options for relevance, value, and autonomy. Self-directed pathways, choice in practice activities, and transparent learning goals increase learner investment across all ability levels.

UDL in Practice — What I Build

  • Tab order set explicitly for every interaction — not left to Storyline defaults
  • Colour combinations tested at 4.5:1 contrast minimum before any content is added
  • All informational images have descriptive alt text; decorative images use empty alt
  • All video content has captions; audio content has transcripts
  • Keyboard-operable alternatives for every drag-and-drop, click interaction
  • Focus indicators visible and logical throughout all course navigation
4C/ID
Four-Component Instructional Design · Jeroen van Merriënboer

Van Merriënboer's 4C/ID model is designed for teaching complex, real-world skills that cannot be broken into isolated sub-skills without losing meaning. It's the right framework when you're designing for holistic professional competency — not topic-by-topic content coverage. Widely used in medical education, technical training, and professional development.

Best For
Complex professional skills, medical and technical training, any learning goal requiring holistic competency rather than discrete knowledge units
Key Difference
Unlike ADDIE, 4C/ID refuses to decompose skills into isolated topics. The whole task is always present in the learning design.
1
Learning Tasks
Authentic, whole-task practice situations that represent real-world complexity. Learners always work with the complete task — not isolated sub-components. Tasks are sequenced from simple to complex while maintaining authenticity throughout.
2
Supportive Information
Information that helps learners perform the non-routine, problem-solving aspects of the task — mental models, case studies, system overviews. Presented before tasks, not embedded within them.
3
Procedural Information
Just-in-time information for the routine, rule-based steps within a task — available exactly when needed during practice, then gradually faded as automaticity develops.
4
Part-Task Practice
Extensive repetitive practice for specific sub-skills that require high fluency — but only after the whole task has been encountered. Not isolated drilling before context.
TPACK
Technological Pedagogical Content Knowledge · Mishra & Koehler

TPACK is the framework for understanding how technology, pedagogy, and subject matter knowledge interact in the design of effective learning. It's particularly valuable when deciding which technology to use and why — not just what's available, but what combination of tools, content, and teaching approach will produce the best learning outcome.

Best For
Technology selection decisions, evaluating authoring tool fit, justifying technology choices to stakeholders, eLearning strategy
Key Insight
Technology alone doesn't improve learning. Pedagogy alone doesn't scale. Content alone doesn't engage. The intersection of all three is where effective eLearning lives.
CK
Content Knowledge
Deep understanding of the subject matter — the facts, concepts, processes, and frameworks that constitute the learning domain. Without this, no technology or pedagogy can compensate.
PK
Pedagogical Knowledge
Understanding of how learning happens — instructional strategies, cognitive load management, assessment design, feedback principles. The science of teaching regardless of technology.
TK
Technological Knowledge
Understanding of current technologies and their capabilities — authoring tools, LMS platforms, AR/VR systems, xAPI — and how they change and evolve over time.
✦
TPACK — The Intersection
The sweet spot where content knowledge, pedagogical knowledge, and technological knowledge overlap. This is where innovative, effective, and appropriate eLearning solutions live — and it's what separates a learning designer from a tool operator.

Which framework for which job?

Quick reference. These are not mutually exclusive — most real projects combine two or three frameworks at different levels of design.

ADDIE
Structured process
Formal corporate training, regulated industries, defined scope, documentation required.
SAM
Agile iteration
Fast timelines, evolving requirements, SMEs who react better to prototypes.
Bloom's Taxonomy
Objective writing
Writing any learning objective, designing any assessment, aligning cognitive demand.
Gagné's 9 Events
Lesson structure
Structuring a single module, procedural training, ensuring no learning step is skipped.
Merrill's Principles
Problem-centred
Experienced learners, complex skills, performance-critical training where relevance must be immediate.
Kirkpatrick
Evaluation strategy
Measurement strategy, demonstrating ROI, business stakeholder conversations about L&D value.
Action Mapping
Behaviour change
Compliance training, any project where performance gap (not knowledge gap) is the real issue.
UDL
Accessibility
ADA/508 required, diverse learner populations, building reusable accessible templates.
4C/ID
Complex competency
Holistic professional skill development, medical/technical training, whole-task learning design.
TPACK
Technology selection
Deciding which technology to use and why, eLearning strategy, justifying tool choices.
